### Traditional Deployment (แบบดั้งเดิม)
เป็นการติดตั้งและรันแอพพลิเคชั่นโดยตรงบนระบบปฏิบัติการของเครื่องเซิร์ฟเวอร์จริง (Physical Server) เช่น ติดตั้ง PHP, Python, MySQL และรันแอพโดยตรง

![รูปภาพสถาปัตยกรรมแบบดั้งเดิม](../sources/images/002/1.png)

อธิบายแต่ละส่วน
- **Hardware** เป็นโครงสร้างพื้นฐาน (infrastructure) หรือ เครื่องคอมพิวเตอร์จริงสำหรับนำมาใช้รันโปรแกรม เช่น CPU, RAM, DISK
- **OS (Operating System)** ระบบปฏิบัติการที่จะทำให้ hardware และ software ต่าง ๆ สามารถทำงานร่วมกันได้
- **Application** เป็น software ที่ทำงานอยู่บนระบบปฏิบัติการ เช่น Web Server, Database หรือแอพพลิเคชันอื่น ๆ ที่เรานำไปติดตั้ง

**ข้อเสีย**

เมื่อบนระบบของเราต้องการที่จะรันแอพพลิเคชันหลายแอพ แต่ว่าแอพพลิเคชันที่เรานำมารันไม่ได้ใช้ library เวอร์ชันเดียวกัน จะทำให้ไม่สามารถนำแอพพลิเคชันมารันบนเครื่องเดียวกันได้ เช่น

- application a ต้องการ python 3.8
- application b ต้องการ python 3.11

จากความแต่งต่างของเวอร์ชัน python ทำให้ทั้ง 2 แอพพลิเคชันไม่สามารถทำงานร่วมกันได้

**อัพเดทซอฟต์แวร์สามารถทำได้ยากมาก** หากมีซอฟต์แวร์หลายตัวบนเครื่องเดียว เพราะการอัพเดทซอฟต์แวร์ตัวหนึ่ง อาจไปกระทบซอฟต์แวร์ตัวอื่น ๆ จนซอฟต์แวร์ที่ได้รับผลกระทบไม่สามารถทำงานได้

*Isolation คือ การแยกทรัพยากรณ์และสภาพแวดล้อมของแต่ละซอฟต์แวร์ออกจากกัน เพื่อให้แอพพลิเคชันสามารถทำงานได้อย่างอิสระและไม่กระทบต่อกัน*

**ขยายระบบยาก** โดยหากต้องการเพิ่มความสามารถของเซิร์ฟเวอร์ให้รองรับจำนวนการใช้งานได้มากขึ้น จำเป็นต้องซื้อเครื่องเซิร์ฟเวอร์และทำการติดตั้งระบบใหม่ทั้งหมด

**ด้านความปลอดภัย** ต่ำมาก โดยหากซอฟต์แวร์ตัวใดตัวหนึ่งบนเครื่องเซิฟเวอร์ถูกเจาะระบบ ผู้ไม่หวังดีสามารถเข้าถึงซอฟต์แวร์ตัวอื่น ๆ ที่อยู่บนเครื่องเดียวกันได้
